# REQUIREMENTS (需求输入模板)

## Role (角色)
- Primary source for delivery requirements. (作为交付需求的主来源)
- Define requirement details, scope boundaries, constraints, and acceptance. (定义需求细节、范围边界、约束、验收标准)

## Delivery Context (交付上下文)
- Product type. (产品类型)
- Delivery mode. (交付模式)
- Target output docs. (目标落盘文档)
- Current repo state. (当前仓库状态)

## Requirements (需求)
- Core delivery requirements. (核心交付需求)

## Scope In (范围内)
- Required deliverables. (必须交付内容)

## Scope Out (范围外)
- Excluded items. (明确排除内容)

## Constraints (约束)
- Technical, platform, or delivery constraints. (技术、平台或交付约束)

## User Scenarios (用户场景)
- Primary scenarios. (主要用户场景)

## Acceptance Criteria (验收标准)
- Observable acceptance checks. (可观察验收检查项)

## Notes (备注)
- Extra implementation or verification notes. (补充实现或验证说明)
