# AGENT_WORK_DIARY (Agent 工作日记)

Rule:
- Append one entry per event. (每次事件追加一条记录)
- Events: phase_start | blocker | difficulty | resolution | review | phase_done. (事件类型)
- Every notable resolution includes a traceability update note. (每个重要解决项都应包含追踪更新说明)
- New `Time` values must start with `YYYY-MM-DD HH:MM:SS.mmm`; an optional short label may follow after ` | `. (新的时间字段必须以该格式起始，后面可选追加简短标签)
- Keep entries short; read only the latest 20 entries by default. (记录保持简短，默认只读取最近 20 条)

## Entry (记录项)
- Time: `YYYY-MM-DD HH:MM:SS.mmm | short-label`. (时间)
- Event: `phase_start|blocker|difficulty|resolution|review|phase_done`. (事件类型)
- Phase: `startup|product_requirements|system_requirements|software_requirements|system_design|detailed_design|implementation|unit_test|integration_test|system_test|acceptance_validation|release_retro`. (阶段)
- Role view: `author|reviewer|tester|approver`. (角色视角)
- Feature or component. (功能或组件)
- Difficulty. (遇到的困难)
- Resolved: `yes|no`. (是否解决)
- Solution. (解决方案)
- Review decision: `pass|fail|conditional|n/a`. (评审结论)
- Traceability update. (追踪更新说明)
- Knowledge review: `created|updated|skipped_with_reason`. (经验沉淀结论)
- Knowledge ref or skip reason. (经验文件引用或跳过原因)
- Structure change: `none|created`. (结构变更)
- Structure ref or reason. (结构文件引用或变更原因)
- Structure contract check: `pass|exception`. (目录结构约束检查)
- Next action. (下一步)
- Evidence. (证据)
