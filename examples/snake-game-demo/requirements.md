- Primary source for delivery requirements. (作为交付需求的主来源)
- Define requirement details, scope boundaries, constraints, and acceptance. (定义需求细节、范围边界、约束、验收标准)
- If `README.md` and this file conflict, this file wins. (若与 `README.md` 冲突，以本文件为准)
- Follow `project_config.yml` for persisted outputs only; phases still execute. (按 `project_config.yml` 决定落盘产物；阶段本身仍执行)

## Delivery Context (交付上下文)
- Product type: `web`. (产品类型为 `web`)
- Delivery mode: single-agent flow. (交付模式为单 agent)
- Target output docs: `design`, `test_report`, `acceptance`. (目标落盘文档)
- Current repo state: product code may be absent; agent may need to create the initial web files. (当前仓库可能还没有产品代码，agent 需要创建初始 Web 文件)

## Requirements (需求)
- Build a playable web Snake game. (开发一个可玩的 Web 贪吃蛇游戏)
- Support keyboard control for snake movement. (支持键盘控制蛇移动)
- Support manual difficulty adjustment that changes the snake speed. (支持手动调节难度，并改变蛇的移动速度)
- Show score and running status in the UI. (在界面中显示分数和运行状态)
- Support start, pause/resume, restart, and game over states. (支持开始、暂停/继续、重开、游戏结束状态)
- The main gameplay UI should fit within a single screen in common desktop browser view without requiring vertical page scrolling. (在常见桌面浏览器视口下，主游戏界面应可单屏完整显示，无需上下滚动页面)
- The layout should adapt to different screen sizes, including common desktop, tablet, and mobile browser views. (布局应适配不同屏幕尺寸，包括常见桌面、平板和手机浏览器视口)

## Scope In (范围内)
- Single-page browser game. (单页浏览器游戏)
- Fixed grid board. (固定网格棋盘)
- Snake movement, food spawning, growth, wall collision, self collision. (蛇移动、食物生成、增长、撞墙、自撞)
- Multiple selectable difficulty levels with different game speeds. (支持多个可选难度等级，并对应不同游戏速度)
- Score display and basic status display. (分数显示和基础状态显示)
- Single-screen-first layout for the main game area in common desktop browser view. (面向常见桌面浏览器视口的主游戏区单屏优先布局)
- Responsive layout adaptation for common desktop, tablet, and mobile screen sizes. (面向常见桌面、平板和手机屏幕尺寸的响应式布局适配)
- Basic manual verification and test report. (基础手动验证和测试报告)

## Scope Out (范围外)
- User account or login. (账号或登录)
- Online leaderboard. (在线排行榜)
- Multiplayer mode. (多人模式)
- Backend service or database. (后端服务或数据库)
- Mobile app packaging. (移动 App 打包)
- Audio system or advanced animation polish. (音效系统或高级动画打磨)

## User Scenarios (用户场景)
- User opens the page and starts the game. (用户打开页面并开始游戏)
- User controls the snake with keyboard input. (用户通过键盘控制蛇)
- User adjusts the difficulty to play at a preferred speed. (用户根据想要的速度调整难度)
- User gains score by eating food. (用户通过吃食物得分)
- User pauses and resumes during play. (用户在游戏中暂停和继续)
- User loses on collision and can restart quickly. (用户碰撞失败后可快速重开)

## Acceptance Criteria (验收标准)
- Board renders correctly in browser. (浏览器中棋盘渲染正确)
- Keyboard control changes snake direction correctly. (键盘控制能正确改变方向)
- Difficulty selection updates the active speed correctly. (难度切换后应正确更新当前游戏速度)
- Eating food increases snake length and score. (吃食物后蛇长度和分数增加)
- Wall collision and self collision both end the game. (撞墙和自撞都会结束游戏)
- Pause/resume works without breaking game state. (暂停/继续不会破坏游戏状态)
- Restart resets score, snake, food, and status. (重开会重置分数、蛇、食物和状态)
- In common desktop browser view, the main gameplay interface is fully visible on one screen without vertical scrolling. (在常见桌面浏览器视口下，主游戏界面单屏完整可见，无需纵向滚动)
- In common desktop, tablet, and mobile browser views, the layout remains usable and readable without major overlap, clipping, or horizontal overflow. (在常见桌面、平板和手机浏览器视口下，布局应保持可用和可读，不出现明显重叠、裁切或横向溢出)

## Notes (备注)
- Default implementation may use plain HTML/CSS/JS unless design phase decides otherwise. (默认可使用原生 HTML/CSS/JS，除非设计阶段另有决定)
- Gameplay should stay simple and stable rather than feature-heavy. (优先简单稳定，不追求功能堆叠)
- Keyboard input should feel responsive during quick direction changes. (快速切向时键盘输入应保持足够灵敏)
- Human-friendly layout and readability matter; avoid forcing users to scroll just to see the full core game area. (应重视人性化布局与可读性，避免用户为了看完整核心游戏区域而被迫滚动页面)
- Responsive behavior should be considered part of usability, not optional polish. (响应式适配属于可用性要求，而不是可有可无的额外美化)
- Verification can use manual checklist if no cheap automated harness is available. (没有低成本自动化时可用手动清单验证)
